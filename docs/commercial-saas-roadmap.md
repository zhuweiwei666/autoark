# AutoArk 商用 SaaS 改造路线图

更新时间：2026-06-01

## 目标

把 AutoArk 从“内部投放工具”改造成可对外交付、可按组织售卖、可排障、可扩容、可审计的广告运营 SaaS。对标 XMP / TopTou 时，不追求界面相似，而是追求客户能稳定完成授权、资产同步、批量建广告、数据分析、审核追踪和团队协作。

## 商用硬门槛

1. 多租户隔离：用户、组织、广告账户、素材、包、任务、Token、统计数据不能串。
2. 客户自助接入：客户能看到接入进度、授权状态、账户/素材/任务是否就绪。
3. 套餐与额度：组织有 plan、billing status、成员数、账户数、素材数、任务数和并发额度。
4. Facebook 商用闭环：公开 OAuth、资产同步、Page/Pixel/账户选择、草稿校验、任务发布、审核追踪。
5. 运营可观测：健康检查、任务失败原因、请求 ID、审计日志、部署验证和迁移脚本。
6. 可管理：超级管理员能管理组织、用户、账户池、App、权限状态；组织管理员能管理本组织成员和资产。
7. 可扩展：后续能接入计费、工单、项目权限、客户级数据看板和 SLA。

## 当前第一批已落地

- 商用安全硬化：租户隔离、OAuth signed state、敏感日志脱敏、全局接口收口。
- 生产迁移脚本：按组织替换唯一索引，补齐可确定归属的数据。
- 商用中心：统一展示组织 readiness、套餐额度、授权/账户/素材/任务闭环和上线保护项。

## 下一批优先级

### P0：客户交付稳定性

- 给批量建广告每个步骤加标准化错误码和客户可读建议。
- 任务失败页展示 Meta 原始错误、AutoArk 解释、建议动作和可重试范围。
- OAuth 授权后自动跑资产诊断：广告账户、Page、Pixel、Business 权限。
- 给任务发布加预算/国家/Page/Pixel 预检，减少发起后失败。

### P1：真正 SaaS 化

- 组织套餐配置页：升级/降级、额度覆盖、功能开关。
- 审计日志：登录、授权、创建广告、删除素材、改组织配置、改用户权限。
- 邀请制成员加入：组织管理员发邀请，成员设置密码后进入组织。
- 项目/客户资产空间：把广告账户、素材、文案包、创意组归到项目，接近 TopTou 的 project 管理。

### P2：对标投放平台体验

- 创建广告流程保存模板：国家、预算、版位、文案、素材组合可复用。
- 账户健康度：余额、风控、禁用、支付问题、近期失败率。
- 素材表现闭环：素材 -> 广告 -> 花费/ROAS/审核状态 -> 推荐复用。
- 批量任务队列：预计创建数量、限流状态、失败重试、并发控制。

### P3：商业运营

- Stripe/人工收款双模式计费底座。
- 客户支持中心：错误截图、任务 ID、授权账号、账户状态一键打包。
- SLA 监控：API、队列、Meta 授权、同步延迟、任务成功率。
- 数据导出：客户级日报、账户日报、素材日报。

## 验收标准

- 新客户不需要工程师介入，也能完成组织登录、Facebook 授权、选择账户/Page/Pixel、上传素材、创建小预算任务。
- 普通客户账号看不到其他组织的任何 Token、账户、素材、任务、统计或 App 配置。
- 每个失败任务都有明确原因和下一步建议。
- 超级管理员能在 5 分钟内判断客户卡在哪一步。
- 发布后脚本能验证主站、Agent、API health、登录、关键业务接口鉴权。
